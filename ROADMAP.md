# Kairos Operator — Roadmap

## Current: v2.0.0 (Released)

All features through v1.1.0 plus:

### Core Improvements
- [x] Route wildcard generation fix (auto-creates Route on KairosConsole instantiation)
- [x] OperatorHub-compliant bundle structure (community-operators-prod + k8s-operatorhub)
- [x] OLM CSV with specDescriptors, statusDescriptors, and proper RBAC
- [x] Scorecard test configuration
- [x] Predictive scaling framework (time-series forecasting interfaces)
- [x] GitOps mode: generate PRs instead of direct SSA patches
- [x] FinOps dashboard integration (cost per namespace/team)
- [x] Custom metric adapter interfaces (Datadog, New Relic, Splunk)
- [x] Multi-cluster policy sync via annotations (OCM-ready)
- [x] SLA-aware scheduling (priority class integration)
- [x] HPA advisor mode (Kairos recommends, HPA executes)

### Bundle & Distribution
- [x] `bundle/` directory compliant with operator-framework standards
- [x] `bundle.Dockerfile` for building OLM bundle images
- [x] `bundle/metadata/annotations.yaml` with proper channel configuration
- [x] CRDs include `spec.validation.openAPIV3Schema` (generated by controller-gen)
- [x] CSV includes `replaces: kairos-operator.v1.1.0`
